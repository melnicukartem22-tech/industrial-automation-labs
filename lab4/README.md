## Мета роботи
Ознайомитися з принципами розробки користувальницьких інтерфейсів (Dashboard) для систем промислової автоматизації та моніторингу за допомогою середовища візуального програмування Node-RED. Набути практичних навичок роботи з локальним та віддаленим репозиторіями Git через інтегровані інструменти керування проєктами (Project History), налаштування віджетів керування, відображення динамічних даних та табличних масивів.

## Хід роботи
[Завантажити flows.json](flows.json)

### Частина 1. Налаштування середовища та ініціалізація проєкту в Git
1. Запустив сервер Node-RED через командний рядок Windows за допомогою команди `node-red`.
2. Перейшов у провіднику за шляхом `%userprofile%\.node-red` для перевірки файлів конфігурації. Налаштував функціонал проєктів у файлі `settings.js`, активувавши параметр `projects: { enabled: true }`.
3. Увійшов в інтерфейс Node-RED через браузер за адресою `http://localhost:1880`. Створив новий локальний проєкт та ініціалізував репозиторій Git для контролю версій розроблюваного потоку даних (Flow).

### Частина 2. Створення базового інтерфейсу та індикаторів
1. За допомогою палітри віджетів `@flowfuse/node-red-dashboard` (Dashboard 2.0) створив нову сторінку з назвою `Widgets` та URL-адресою `/widgets`.
2. Додав групу елементів для відображення числових значень.
3. Створив три кругові індикатори типу `ui-gauge` для візуалізації умовної швидкості обертання двигунів або виконавчих механізмів. Налаштував сектори колірної індикації: зелений (нормальний режим), жовтий (попереджувальний), червоний (критичний аварійний рівень).

### Частина 3. Модифікація елементів керування та динамічна логіка
1. Налаштував блок повзунка `ui-slider` (Slider) у рамці `change number`. Перевів режим відправки даних у стан **"While sliding"** (відправка значень під час руху), що дозволило системі реагувати на дії користувача безперервно в реальному часі, без очікування відпускання кнопки миші.
2. Модифікував випадаючий список `ui-dropdown` (Select Option), додавши новий варіант вибору: `Label: Lab 4`, `Value: Done!`.
3. Створив новий віджет індикатора `ui-gauge`, перевів його тип у режим **Tile (Плитка)** та присвоїв назву `Status`. 
4. Вирішив проблему інтеграції віджета: підключив його вхідний порт до виходу вузла `delay` у рамці автоматичного генерування чисел. Налаштував властивість зміни кольору фону (Fill background) залежно від діапазону вхідного payload (0-3 — сірий/зелений, 4-7 — жовтий, 8-10 — червоний).

### Частина 4. Робота з таблицями (ui-table)
1. Створив нову сторінку Dashboard для моніторингу масивів даних: `Name: Data`, `Path: /pagedata`, `Icon: table`.
2. Імпортував готовий JSON-код структури таблиці. Усунув конфлікт дублювання сторінок з однаковими URL-адресами шляхом переприв'язки конфігураційної групи `DataGroup1` до створеної вручну сторінки `Data`.
3. Модифікував структуру вузла `ui-table`, додавши нову колонку:
   * **Title:** `Num Value`
   * **Key:** `num`
   * **Type:** `text` (відображення поточного обробленого числа).
4. Змінив логіку генератора масивів у вузлі `template`, додавши відповідне поле `"num": {{progress1}}` та `"num": {{progress2}}` всередину об'єктів структури JSON.

## Аналіз отриманих результатів та висновки
Під час виконання лабораторної роботи було успішно побудовано багатовіконну панель керування (Dashboard 2.0) в середовищі Node-RED, адаптовану для задач операторського контролю (SCADA/HMI). 

Налаштовані зв'язки між елементами керування (`ui-slider`, `ui-dropdown`) та індикаторами відображення (`ui-gauge` типу Tile) продемонстрували високу швидкість обробки подій та низьку затримку при передачі повідомлень через об'єкт `msg.payload`. Модернізація табличної структури `ui-table` дозволила структурувати вивід динамічних даних у зручному для аналізу форматі. 

Додатково було освоєно життєвий цикл розробки програмного забезпечення за допомогою вбудованого інструментарію Git в Node-RED, що дозволяє ефективно керувати версіями промислових скриптів, фіксувати зміни та оперативно усувати помилки в архітектурі потоків даних. Всі поставлені завдання виконано в повному обсязі.
