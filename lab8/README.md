#### Таблиця 1. Пояснення значень заголовків запиту (Request Headers)
| Заголовок | Значення | Пояснення |
| :--- | :--- | :--- |
| **Accept** | `text/html,application/xhtml+xml,application/xml;q=0.9,*/*;q=0.8` | Перелік форматів контенту, які браузер здатний зрозуміти та відобразити користувачу. |
| **Accept-Encoding** | `gzip, deflate, br, zstd` | Перелік алгоритмів стиснення даних, які підтримує браузер для прискорення завантаження. |
| **Accept-Language** | `uk-UA,uk;q=0.9,en-US;q=0.8,en;q=0.7` | Мовні уподобання користувача. Найвищий пріоритет (`q=0.9`) має українська мова. |
| **Alt-Used** | `barcatreader.onrender.com` | Вказує альтернативний хост, який використовується для поточного мережевого з'єднання. |
| **Cache-Control** | `no-cache` | Інструкція для проміжних серверів та кешу не використовувати старі копії, а зробити прямий запит. |
| **Connection** | `keep-alive` | Інструкція утримувати поточне TCP-з'єднання відкритим для наступних запитів. |
| **Host** | `barcatreader.onrender.com` | Доменне ім'я сервера (та порт), до якого спрямовується HTTP-запит. |
| **Pragma** | `no-cache` | Застарілий заголовок епохи HTTP/1.0, що дублює функцію Cache-Control для сумісності. |
| **Upgrade-Insecure-Requests** | `1` | Сигнал від браузера про бажання автоматично перейти з HTTP на захищений протокол HTTPS. |
| **User-Agent** | `Mozilla/5.0 (Windows NT 10.0; Win64; x64; rv:151.0) Gecko/20100101 Firefox/151.0` | Повна технічна інформація про операційну систему клієнта (Windows 11) та версію браузера (Firefox 151). |

#### Таблиця 2. Пояснення значень заголовків відповіді (Response Headers)
| Заголовок | Значення | Пояснення |
| :--- | :--- | :--- |
| **alt-svc** | `h3=":443"; ma=86400` | Сервер повідомляє, що підтримує роботу за швидшим протоколом HTTP/3 (на порті 443). |
| **content-encoding** | `br` | Тіло відповіді було стиснуто сервером за допомогою сучасного алгоритму Brotli. |
| **content-length** | `1070` | Точний розмір корисного контенту відповіді (HTML-коду) в байтах. |
| **content-type** | `text/html; charset=utf-8` | Формат надісланих даних (веб-сторінка HTML) та кодування тексту (стандарт UTF-8). |
| **date** | `Mon, 08 Jun 2026 12:34:43 GMT` | Точний час та дата генерації відповіді безпосередньо вебсервером. |
| **server** | `cloudflare` | Вказує, що запит обслуговується через захисну мережу та проксі Cloudflare. |
| **x-render-origin-server**| `Kestrel` | Специфічний заголовок хостингу Render, який показує оригінальний сервер програми (.NET Kestrel). |

## Висновок
Під час виконання даної лабораторної роботи було детально досліджено принципи функціонування прикладного протоколу HTTP (версій 1.1 та 3) за допомогою інструментів розробника (Network Monitor) браузера Firefox та візуального середовища Node-RED. 

На практиці освоєно архітектурну різницю між методами `GET` (запит даних) та `POST` (відправка даних на сервер), проаналізовано структуру HTTP-заголовків запиту та відповіді, а також закріплено навички роботи з типом контенту `multipart/form-data` для передачі бінарних об'єктів. Окрім цього, розроблено автоматизований скрипт для надсилання HTTP-запитів і парсингу DOM-дерева веб-сторінок, та піднято власний локальний вебсервер.
